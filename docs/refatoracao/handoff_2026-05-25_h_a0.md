## 3. Objetivo desta sessão

Implementar **H-A0 hard** no `gerador_csp.py`: quando demanda é `("regiao", R, qtd)` com `R ∈ ANCORAS_POR_REGIAO`, para cada subregião `obrigatoria=True` daquela região, exigir ≥1 slot **dentro do mesmo treino** cuja subregião escolhida seja aquela subregião. Subregiões **não listadas** em `ANCORAS_POR_REGIAO[R]` ficam HARD banidas dos slots dessa demanda (ex: `bracos` banido em slot `("regiao","upper",N)`).

Variável CSP nova: `sub_idx[s] = IntVar(0, len(subs_R)-1)` por slot de demanda região, canalizada via `assign` (mesmo padrão de `bloco_idx[s]` em Fatia 4.A).

Graceful degradation análoga à H-A1: pool sem candidato → constraint pulada com `degraded=True`; conflito de cardinalidade (vagas < n_obrigatórias) → constraint colaborativa `sum(obrig_usada) >= vagas` força N distintas.

Resultado dict ganha `h_a0_aplicadas: list[dict]` análogo a `h_a1_aplicadas`.

## 4. Decisões já fechadas (NÃO reabrir sem motivo forte)

Decisões tomadas na sessão 2026-05-25 entre Bernardo + Claude após apresentação da spec inicial. Cada uma fechada explicitamente.

### 4.1 Agregação **per-treino** (não cross-treino)

Diferente da H-A1 que agrega cross-treino. Justificativa: semântica de "treino de upper" é "cobre upper NAQUELE treino", não "alguma hora da rotina". Achado de hoje confirma: T1 sem peito + T2 com peito passaria num gate cross-treino, mas T1 continua sendo ruim de fazer.

**Implementação**: agrupar `slot_ids` por `(t_idx, R)` em vez de só por `R`. Constraint H-A0 itera por tupla — uma constraint por (treino, região) com âncoras obrigatórias.

### 4.2 Interação H-A0 × H-A1: **caminho (b) — marker no slot**

Quando H-A0 obriga ≥1 slot de subregião X dentro de demanda região R, esse slot deve ATIVAR H-A1[X] (ex: slot virou de subregião peito → H-A1 exige `empurrar_compostos` obrigatório).

**Implementação**: slot ganha campo `subregiao_obrigada_por_ha0: str | None` na estrutura de grupos. H-A1 lê esse campo. Quando presente, H-A1 ativa pra aquele slot como se a demanda fosse `("subregiao", X, _)`.

Caminho (c) "Aderência + softmax cobre" foi rejeitado: é o tipo exato de soft que falha quando aparece o buraco real (mesmo padrão do antipadrão que estamos fechando).

### 4.3 Rejeição de subregiões não-âncora: **HARD (Caminho A)**

Subregiões em `REGIAO_PARA_SUBREGIOES[R]` mas **não listadas** em `ANCORAS_POR_REGIAO[R]` (ex: `bracos` em upper, `adutores` em lower) ficam banidas hard dos slots dessa demanda região.

**Implementação**: pra cada slot `s` de demanda `("regiao", R, qtd)`, na canalização do `sub_idx[s]` via assign, candidatos cujo `ex.subregiao` não está em `[a["subregiao"] for a in ANCORAS_POR_REGIAO[R]]` têm `assign[(s, cidx)] == 0` forçado (ou são removidos do `pool_slot[s]` upstream — escolher o caminho mais limpo na implementação).

**Importante**: essa rejeição é **só dentro do escopo dos slots de demanda região R**. Outras demandas na mesma rotina (ex: `("padrao", "biceps", 1)`) continuam livres — usuário pode pedir `("regiao","upper",2) + ("padrao","biceps",1)` e ter braços, só não como acidente dentro do upper.

### 4.4 Reuso de `ANCORAS_POR_REGIAO` existente

Constante já existe em `gerador_treino.py:142` com a tabela curada. Importar com `from gerador_treino import ANCORAS_POR_REGIAO` (mesmo padrão de `ANCORAS_POR_SUBREGIAO`, `PADRAO_PARA_SUBREGIAO`, `REGIAO_PARA_SUBREGIOES`). **NÃO duplicar a tabela no CSP**.

### 4.5 NÃO importar `_decompor_demanda_regiao` do antigo

Pode ser consultada pra entender semântica clínica (Hamilton + filtro de obrigatoriedade), mas reusar código viola norte Seção 5 (declarativo até o fim) e antipadrão Seção 7. H-A0 reescreve a regra como constraint declarativa.

### 4.6 NÃO modelar `PROPORCAO_COMPOSTOS = 0.6` nesta frente

Cobertura de compostos vem via H-A1[X] (que ativa por marker, decisão 4.2) — peito Principal cai em `empurrar_compostos` naturalmente. Se empírico mostrar que não cobre suficiente, abre frente separada (provavelmente S-A1).

### 4.7 ConfigVariedade preservado

H-A0 não muda nada no wiring de variedade (top-K + softmax). Continua default `ConfigVariedade()` em `gerar_rotina_csp` pra `/gerar` (E.1).

### 4.8 Nomenclatura: **H-A0**, não H-A2

Sufixo `0` indica "âncora mais primitiva" (decide subregião antes de H-A1 decidir padrão dentro da subregião). Ordem cronológica de decisão semântica. Roadmap já atualizado.

## 6. Spec técnica (rascunho de implementação)

### 6.1 Variável CSP nova

```python
# Em _construir_modelo, depois do laço que cria assign[(sid, cidx)]:
# Cria sub_idx[s] pra cada slot s de demanda nível regiao com R em ANCORAS_POR_REGIAO.

sub_idx: dict[int, cp_model.IntVar] = {}
subs_por_regiao: dict[str, list[str]] = {}  # R -> ordenação estável

for grupos_t in treinos:
    for g in grupos_t:
        nivel, escopo, _ = g["demanda"]
        if nivel != "regiao" or escopo not in ANCORAS_POR_REGIAO:
            continue
        subs_ancora = [a["subregiao"] for a in ANCORAS_POR_REGIAO[escopo]]
        subs_por_regiao[escopo] = subs_ancora
        for sid in g["slot_ids"]:
            sub_idx[sid] = model.NewIntVar(0, len(subs_ancora) - 1, f"sub_idx_{sid}")
```

### 6.2 Canalização via assign (Caminho A da decisão 4.3)

```python
# Para cada slot s de demanda regiao R, para cada cidx do pool_slot[s]:
for sid, info in slot_por_sid.items():
    nivel, escopo, _ = info["demanda"]
    if nivel != "regiao" or escopo not in ANCORAS_POR_REGIAO:
        continue
    subs_ancora = subs_por_regiao[escopo]
    for cidx, ex in enumerate(info["pool_slot"]):
        if ex.subregiao in subs_ancora:
            sub_id_local = subs_ancora.index(ex.subregiao)
            model.Add(sub_idx[sid] == sub_id_local).OnlyEnforceIf(assign[(sid, cidx)])
        else:
            # HARD BAN: subregião não é âncora dessa região
            model.Add(assign[(sid, cidx)] == 0)
```

**Alternativa mais limpa**: filtrar `pool_slot[sid]` upstream pra remover candidatos com `ex.subregiao not in subs_ancora`. Decisão pendente 5.1.

### 6.3 Constraint H-A0 hard per-treino

```python
h_a0_aplicadas: list[dict] = []

# Agregação per-treino (decisão 4.1): chave = (t_idx, R)
slots_por_treino_regiao: dict[tuple[int, str], list[int]] = defaultdict(list)
for t_idx, grupos_t in enumerate(treinos):
    for g in grupos_t:
        nivel, escopo, _ = g["demanda"]
        if nivel == "regiao" and escopo in ANCORAS_POR_REGIAO:
            slots_por_treino_regiao[(t_idx, escopo)].extend(g["slot_ids"])

for (t_idx, R), sids in slots_por_treino_regiao.items():
    if not sids:
        continue
    subs_ancora = subs_por_regiao[R]
    obrigatorias = [a["subregiao"] for a in ANCORAS_POR_REGIAO[R] if a["obrigatoria"]]

    # Detecta degraded-por-pool: subregião obrigatória sem nenhum candidato no pool dos slots
    ativas: list[str] = []
    degradadas_pool: list[str] = []
    for sub in obrigatorias:
        tem_candidato = any(
            ex.subregiao == sub
            for sid in sids
            for ex in slot_por_sid[sid]["pool_slot"]
        )
        (ativas if tem_candidato else degradadas_pool).append(sub)

    for sub in degradadas_pool:
        h_a0_aplicadas.append({
            "treino": t_idx, "regiao": R, "subregiao_obrigatoria": sub,
            "n_slots": len(sids), "degraded": True,
            "motivo": "pool sem candidato (provavelmente filtrado por H-P1)",
        })

    if not ativas:
        continue

    vagas = len(sids)
    n_ativas = len(ativas)

    if vagas >= n_ativas:
        # Caso normal: 1 constraint por subregião obrigatória ativa.
        for sub in ativas:
            sub_id_local = subs_ancora.index(sub)
            bools = []
            for sid in sids:
                b = model.NewBoolVar(f"ha0_{R}_{sub}_{t_idx}_{sid}")
                model.Add(sub_idx[sid] == sub_id_local).OnlyEnforceIf(b)
                model.Add(sub_idx[sid] != sub_id_local).OnlyEnforceIf(b.Not())
                bools.append(b)
            model.Add(sum(bools) >= 1)
            h_a0_aplicadas.append({
                "treino": t_idx, "regiao": R, "subregiao_obrigatoria": sub,
                "n_slots": vagas, "degraded": False,
            })

            # Marker pra H-A1 (decisão 4.2): registra que ≥1 slot DESSE treino
            # nessa região será de subregiao=sub.
            # Implementação concreta — ver decisão pendente 5.2.
            # Opção (a): popular slot_por_sid[sid]["subregiao_obrigada_por_ha0"]
            # = sub (mas qual sid? a BoolVar bools só garante ≥1; não sabemos qual).
            # ATENÇÃO: marker é POR REGIÃO/TREINO, não por slot específico.
            # Sugestão: registrar em estrutura paralela
            # `subregioes_obrigadas_ha0[(t_idx, R)] = set(ativas)` que H-A1
            # lê pra ativar sobre TODOS os slots da demanda região (não só os
            # que efetivamente vão pra X — H-A1 fica ≥1 em qualquer um deles,
            # o que se reduz exatamente à mesma garantia).
    else:
        # Conflito de cardinalidade (vagas < n_obrig ativas)
        obrig_usadas_vars = []
        for sub in ativas:
            sub_id_local = subs_ancora.index(sub)
            usada = model.NewBoolVar(f"ha0_usada_{R}_{sub}_{t_idx}")
            bools_sub = []
            for sid in sids:
                b = model.NewBoolVar(f"ha0_{R}_{sub}_{t_idx}_{sid}")
                model.Add(sub_idx[sid] == sub_id_local).OnlyEnforceIf(b)
                model.Add(sub_idx[sid] != sub_id_local).OnlyEnforceIf(b.Not())
                bools_sub.append(b)
            model.Add(sum(bools_sub) >= 1).OnlyEnforceIf(usada)
            model.Add(sum(bools_sub) == 0).OnlyEnforceIf(usada.Not())
            obrig_usadas_vars.append(usada)
        model.Add(sum(obrig_usadas_vars) >= vagas)
        motivo = f"conflito_cardinalidade vagas={vagas}<n_obrig_efetivo={n_ativas}"
        for sub in ativas:
            h_a0_aplicadas.append({
                "treino": t_idx, "regiao": R, "subregiao_obrigatoria": sub,
                "n_slots": vagas, "degraded": True, "motivo": motivo,
            })
```

### 6.4 Marker pra H-A1 (decisão 4.2 + pendente 5.2)

Após o bloco H-A0, o bloco H-A1 atual itera sobre `slots_subregiao_explicita` (demanda nível subregião). Estender pra também iterar sobre `subregioes_obrigadas_ha0[(t_idx, R)] = set([sub, ...])` — pra cada `sub` ali, agregar os `slot_ids` da demanda região correspondente como se fossem demanda `("subregiao", sub, _)`.

Detalhe: H-A1 vai exigir ≥1 slot com `padrao == empurrar_compostos` entre os slots da demanda região upper (treino t). Como H-A0 garante ≥1 slot com `subregiao == peito` no mesmo conjunto, e empurrar_compostos só existe em peito (via PADRAO_PARA_SUBREGIAO), a interseção é viável.

**Validar empiricamente** essa interação após implementação — se solver entrar em conflito interno (ex: H-A1 + H-A0 + cargas + S-T1), pode aparecer `INFEASIBLE` em casos extremos. Pra esses casos, retry de degradação pode ser necessário (não nesta frente; registrar como achado se aparecer).
