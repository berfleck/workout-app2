## 1. Por que esta sessão existe (contexto + achado pós-H-A0)

Em 2026-05-25, após mergear H-A0 (cobertura de subregiões obrigatórias por região), Bernardo perguntou: "se o usuário pede `ombro(2)`, o app insere um ombro isolado?". Sondagem empírica em 40 seeds revelou viés sistemático no CSP atual (pós-H-A0) que não estava no motor antigo:

**Tabela comparativa (40 seeds por config, demanda direta `subregiao(2)`):**

| Demanda | Motor antigo | CSP atual (pós-H-A0) |
|---|---|---|
| `ombro(2)` | **100% (ombro_composto + ombro_isolado)** ✓ | 100% (ombro_composto + posterior_ombro) — **0% ombro_isolado** ✗ |
| `perna_posterior(2)` | **52% (hinge + abduction) / 48% (hinge + knee_flexion)** ✓ | 100% (hinge + abduction) — **0% knee_flexion** ✗ |
| `peito(2)` | 100% (composto + isolado) ✓ | 47% (2× composto) / 53% (composto + isolado) (mix razoável) |
| `perna_anterior(2)` | 100% (squat_bi + squat_uni) ✓ | 100% (squat_bi + squat_uni) ✓ (banco com só 2 padrões = trivial) |

**Diagnóstico técnico**: H-A1 (Bloco 2.5) força só padrões `obrigatoria=True`. As demais vagas ficam livres pro solver decidir entre padrões não-obrigatórios. **S-B1 (evitar agonistas no bloco) domina o trade-off** e empurra a vaga sobrando pro padrão "antagonista" da subregião:
- ombro(2): composto = push, isolado = push (mesmo grupo). `posterior_ombro` = pull (antagonista) → solver SEMPRE escolhe posterior pra evitar par agonista.
- perna_posterior(2): hinge = hamstring, knee_flexion = hamstring (mesmo grupo). `abduction` = glute/abdutor (grupo diferente) → solver SEMPRE escolhe abduction.

**Os pesos curados 3/2/1 em `ANCORAS_POR_SUBREGIAO` existem mas o CSP os IGNORA** — só consome `obrigatoria=True`. O motor antigo consumia esses pesos via `_distribuir_quotas_*` e por isso entregava distribuição clinicamente correta.

**Importante — NÃO é "constante histórica"**: validação empírica acima confirma que o antigo entrega ombro_isolado em **100% das rotinas** quando demanda é direta `ombro(2)`. A confusão inicial vinha do relatório E.0 H-A0 que mostrava ombro coberto em apenas 48-52% das rotinas Full Body 2T — mas essa métrica mede "rotina inteira tem ombro?" no contexto de 14 outros slots, **não distribuição em demanda direta**.

**S-A1 é frente correta e fecha REGRESSÃO real do CSP vs antigo.**

## 4. Decisões já fechadas (NÃO reabrir sem motivo forte)

### 4.1 S-A1 condicionado a "vagas > n_obrigatórias na subregião"

Decisão da outra sessão Claude, validada por Bernardo. Razão: isola trade-off com S-B1.

**Implementação**: penalty S-A1 só dispara em slots da subregião que NÃO estão forçados pela constraint hard H-A1. Se demanda é `ombro(2)` com 1 obrigatória (`ombro_composto`), apenas a SEGUNDA vaga recebe penalty S-A1. Em `ombro(1)`, NENHUMA vaga recebe (toda vaga é forçada pelo H-A1).

**Forma técnica sugerida**: por subregião X com âncoras não-obrigatórias e demanda explícita ou via marker H-A0:
- `n_vagas_extras = n_slots(X) - n_obrigatorias_de(X)`
- Se `n_vagas_extras >= 1`: pra cada slot S da subregião e cada candidato C com `padrao` ∈ não-obrigatórias[X], adiciona penalty `assign[(S, C)] * (peso_max - peso_da_ancora(C.padrao)) * peso_sa1` no objetivo.

Quando `n_vagas_extras = 0` (caso degenerado), constraint é trivialmente vazia.

### 4.2 S-B1 fora do escopo S-A1

Considerei "S-B1 está agressivo demais same-subregião" como opção 4 mas descarto. Razão: S-B1 está fazendo o trabalho certo em pares cross-subregião (peito+ombro, costas+braços, etc). Mexer global tem alto risco de regressão. Se S-A1 condicionada não resolver, abre frente própria pra refinar S-B1.

## 5. Pontos de decisão pendentes (perguntar ANTES de codar via AskUserQuestion)

### 5.1 Forma do penalty S-A1: linear, quadrática ou diferença ao peso máximo?

Opções:
- **(a) Linear ao peso**: `pen = (peso_max - peso_ancora) * peso_sa1`. Peso 3 → pen 0, peso 2 → pen 1×peso_sa1, peso 1 → pen 2×peso_sa1. Simples, fácil de calibrar.
- **(b) Quadrática**: `pen = (peso_max - peso_ancora)² * peso_sa1`. Acentua diferença entre 2 e 1, suaviza entre 3 e 2.
- **(c) Inverso direto**: `pen = (1/peso_ancora) * peso_sa1`. Peso 1 → 1×peso_sa1, peso 2 → 0.5×peso_sa1, peso 3 → 0.33×peso_sa1.

Recomendação preliminar: **(a) linear** — mais previsível em calibração, alinha à estrutura simples dos pesos 3/2/1.

### 5.2 Comportamento quando H-A0 marker contribui (demanda nível região)

Hoje, demanda `("regiao", "upper", 6)` → H-A0 força 1 slot peito (composto obrigatório), 1 slot costas, 1 slot ombro (composto obrigatório). Sobra 3 vagas livres em qualquer sub upper.

Pergunta: S-A1 ativa pras vagas sobrando dessa demanda região? Se sim, qual sub conta? (vaga não fica fixa em uma sub específica — sub_idx é variável)

Opções:
- **(a) Sim, condicional via `sub_idx`**: pra cada slot livre, penalty depende da subregião escolhida E do padrão. Mais expressivo mas adiciona variáveis derivadas no modelo.
- **(b) Não, só ativa em demanda nível subregião**: simplifica, mas perde caso de demanda região com vagas grandes.

Recomendação preliminar: **(b) não no MVP** — ataca o achado direto (ombro(2), perna_posterior(2)) sem complexificar. Demanda região fica como pendência futura se Bernardo confirmar gap em uso real.

### 5.3 Calibração inicial — chute pra `peso_sa1` antes de coordinate descent

Métrica: distribuição em demanda direta `ombro(2)` (40 seeds). Alvo: ombro_isolado ≥ 50%.

Faixa inicial sugerida:
- **(a) peso_sa1 = 5**: leve. Pode não dominar S-B1 (peso 10 default).
- **(b) peso_sa1 = 15**: dominante sobre S-B1. Risco: empurrar isolado mesmo quando S-B1 razoavelmente preferiria posterior.
- **(c) peso_sa1 = 8**: meio termo. Equilibra trade-off.

Recomendação preliminar: começar com **(c) 8** + coordinate descent ±50% se não bater alvo.

### 6.2 Bloco S-A1 em `_construir_modelo`

Inserir DEPOIS do bloco H-A1 atual (linha ~898), antes de S-T1.

```python
# ── S-A1 (SOFT): distribuição entre âncoras não-obrigatórias ─────────────
# Consome o `peso` curado em ANCORAS_POR_SUBREGIAO (3/2/1) que H-A0/H-A1
# ignoram. Decisão 4.1 do handoff S-A1: só ativa em slots da subregião
# que NÃO estão forçados pela hard H-A1 (vagas sobrando após obrigatórias).
#
# Forma: pra cada slot livre da subregião X e cada candidato C com padrão
# não-obrigatório de X, penalty = (peso_max - peso_da_ancora(C.padrao)) *
# peso_sa1. Solver minimiza → vaga sobrando prefere padrão de peso alto.
#
# Quando peso_sa1=0 (default), bloco skipado byte-a-byte.
if peso_sa1 > 0:
    for sub, sids_lista in slots_subregiao_explicita.items():
        ancoras_da_sub = ANCORAS_POR_SUBREGIAO.get(sub, [])
        obrigatorias = [a for a in ancoras_da_sub if a.get("obrigatoria")]
        nao_obrigatorias = [a for a in ancoras_da_sub if not a.get("obrigatoria")]
        if not nao_obrigatorias:
            continue  # nada pra distribuir

        n_obrig = len(obrigatorias)
        # Vagas garantidas pela sub (separada de len(sids_lista) — ver fix
        # H-A1 quando marker H-A0 contribui).
        n_vagas = vagas_garantidas_por_sub.get(sub, len(sids_lista))
        n_vagas_extras = n_vagas - n_obrig

        if n_vagas_extras < 1:
            continue  # sem vaga sobrando — H-A1 ocupa tudo

        peso_max = max(a["peso"] for a in nao_obrigatorias)
        peso_por_padrao = {a["padrao"]: a["peso"] for a in nao_obrigatorias}

        # Adiciona penalty pra cada slot da sub × candidato não-obrig.
        for sid in sids_lista:
            slot_info = slot_por_sid[sid]
            for cidx, ex in enumerate(slot_info["pool_slot"]):
                if ex.padrao in peso_por_padrao:
                    custo = (peso_max - peso_por_padrao[ex.padrao]) * peso_sa1
                    if custo > 0:
                        # Penalty proporcional a assign[(sid, cidx)] — 0 ou custo.
                        sa1_pen = model.NewIntVar(
                            0, custo, f"sa1_{sub}_{sid}_{cidx}",
                        )
                        model.Add(sa1_pen == custo * assign[(sid, cidx)])
                        penalidades.append(sa1_pen)
```
